Classes & Objects - Questions

These exercises are designed to expand on the topics introduced in the Classes & Objects chapter of the Java Development training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is to define classes, create objects and work with attributes, properties and instance methods.

Use Logback for writing messages to the console, directly writing to System.out no longer allowed.

  1. Triangles

    Define a class to represent a triangle storing the lengths of the three sides of the triangle. Add methods to indicate if the triangle is:

    • isosceles (two sides of the same length)
    • equilateral (all sides the same length)
    • right angled (sum of the squares of the two shorter sides equals the square of the longest side)

    You should be able to specify the side lengths in any order when creating the triangle (no need to supply them in ascending or descending order). Store the sides in an array of three doubles and sort these so that the longest side is the last entry in the array (this will make checking for right angled triangles easier).

    Throw an IllegalArgumentException to reject an attempt to create an invalid triangle where the sum of the lengths of the shorter sides is less than or equal to the length of the longest side, or any of the sides are zero or negative.

    Provide two constructors, one takes an array of three side lengths as doubles (check there are exactly 3 sides) and one with 3 separate double values for the sides. Use constructor chaining (a this() method call) to implement one of these constructors.

    Test your program with suitable values such as:

    • 2, 3, 4 - plain triangle
    • 3, 3, 2 - isosceles
    • 3, 3, 3 - equilateral
    • 5, 4, 3 - right angled
    • 1, 2, 3 - invalid

  2. Lottery

    Create a class representing a lottery ticket to store a set of 6 unique numbers (create a TreeSet<Integer> to hold the numbers). In the initialisation method generate 6 randomly chosen numbers between 1 and 59 using Random.nextInt().

    Add a method to the class to match this ticket against another ticket and return a List of the matched numbers - you will find the retainAll() method useful for matching the numbers in both tickets. Add a toString() that shows the numbers in the ticket - you don't need any getter or setter methods for this class.

    In a test class create a list of 10 lottery tickets and generate a draw ticket. Print out the draw ticket numbers, and each tickets numbers indicating how many numbers matched - add the string 'WINNER' to any with 3 or more matched numbers.

    Add a second constructor that accepts a list of integers as a parameter and save these numbers in the ticket. Validate the the saved set of numbers is of the correct size (6) and all numbers are within the permitted range (1..59).

    Starting with the following table of planetary data:

    "Jupiter",5.2,142984

    "Mars",1.52,6792
    "Earth",1,12756
    "Venus",0.72,12104
    "Saturn",9.54,120536

    Define a class to store planetary data of name, distance from the sun (double) and diameter (int). Provide a constructor to create a planet and getter methods for each attribute (name, distance and diameter). You can copy and paste from the table to quickly create the parameter list needed to invoke a planet constructor.

    Create a second SolarSystem class that holds a list of individual planets and have its constructor initialise the list of planets. Have this class define a list() method that returns a copy of the list of planets.
