Collections - Questions

These exercises are designed to expand on the topics introduced in the Collections chapter of the Java Development training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

  1. UK Lottery

    A UK lottery ticket entry requires 6 unique numbers between 1 and 59. Write a program to generate one row of random ticket numbers. Do this by:

    1. create an empty list for the numbers
    2. loop round until the list has 6 numbers
    3. use random.randnext() to generate a number between 1 and 59
    4. add this number to the list if the list does not already contain the number
    5. print out the list
    6. (optional) look at the Collections class to work out how to sort the numbers into numerical order before printing the list.

  4. Rock Paper Scissors

    Rock-paper-scissors is a zero sum hand game played between two people. To play, each player simultaneously forms one of three shapes with their hand:

    • rock - fingers curled in a fist
    • paper - fingers outstretched in a flat hand
    • scissors - first and middle fingers outstretched to form a V, other fingers curled

    The shapes are compared so that the winner is detrmined from:

    • rock blunts scissors
    • paper wraps rock
    • scissors cut paper

    In the event of a draw, the players select again and continue until a winner emerges or a player quits.

    Implement your version of the game. Prompt the player for a move (accept either the first letter or full name of the move), generate a random computer move, compare the results and display the winner. If there is a draw then loop round and repeat until the user quits or a winner is found.

    Use a decision matrix to compare the outcome of each game. That is create a 3x3 array and in each cell indicate the winner where the row is selected by the users choice and the column is selected by the computers choice. You will need to map rock, paper and scissors into the numbers 0, 1 and 2 (to index the collection).

  5. Poker School

    Simulate the basic mechanism of a card playing game with a deck of decks and players who are each dealt a hard of cards from the deck. A card can be simulated using a number between 0 and 51. A card's face value will be the card number modulo (remainder) 13 and the suit will be the card number divided by 13 (rank the suits clubs 0, diamonds 1, hearts 2 and spades 3). The card deck and the player's hands will be simulated using ArrayList<Integer> objects.

    There are a number of steps you will need to follow to develop this program:

    1. Create an array list to hold the deck of cards and fill this list with integer values from 0 to 51 (you'll need to use a counted for loop to do this)
    2. Create a class level (private static) method to shuffle the deck of cards. This method should have a List<Integer> parameter which is the deck to shuffle and return void as the list will be modified directly. The shuffle algorithm should loop through every card in the deck and swap it with any other card in the deck: use a random index value between 0 and 51 to select which card to swap with the current card - it is possible and valid for a card to be swapped with itself and therefore not change position within the deck. Call this function to shuffle the deck.
    3. Create a list of players each of which is represented by an empty ArrayList<Integer>. This list (the poker table) will be declared as List<List<Integer>>: use the Arrays.asList() method to initialise this list.
    4. Create a class level (private static) method to deal the deck of cards to the players. You will need to pass in the deck of cards (a list), the list of players (a list of lists), and the number of cards to deal to each player. This method will return void as the deck and players hands passed as parameters will be modified directly. Use a proper dealing algorithm that uses a counted loop to deal n cards one at a time to each player: use an inner iterative loop to deal a card to each player by using the List.remove(0) method to take the next card from the front of the deck and add it to the players hand. Call your deal method to deal 5 cards to each player.
    5. Define another class level method to print out a list of cards. This method should take a string name parameter and a card list (either a deckor a player's hand) and print out the card number as a value and suit. You will find these lookup strings useful for converting the value (number%13) and suit (number/13) to characters: 
      private static final String SUITS = "CDHS";
private static final String VALUES = "A23456789TJQK";
    6. Use your printer function to print out the player's hands and the cards left in the deck. format your output in a similar manner to the following: 
      Doyle     : 4D AD AD 3S 3H
Bill      : AC 3S 2D 2D 2C
Henry     : AH 3S 3D 2H 3H
Deck      : 2S 3C 4S AD 4D 2C 2S 4C ...
    7. Strictly optional and no solution is provided but extend you program to compare the players hands and declare a winner using standard draw poker hand ranking.
