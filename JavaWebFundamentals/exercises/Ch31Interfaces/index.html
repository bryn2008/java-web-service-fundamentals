Interfaces - Questions

These exercises are designed to expand on the topics introduced in the Interfaces chapter of the Java Development training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is to work with classes that implement interfaces and to specify class behaviour through interfaces. Continue to use JUnit test cases to exercise the functionality of the classes you develop.

  1. Sorting Planetary Data

    In a previous chapter your were asked to define a class to represent a planet and another to represent the solar system holding a list of planets initialised with the following data:

    PlanetAU from SunDiameter
    "Mercury",0.39,4879
    "Neptune",30.06,49528
    "Jupiter",5.2,142984
    "Uranus",19.18,51118
    "Mars",1.52,6792
    "Earth",1,12756
    "Venus",0.72,12104
    "Saturn",9.54,120536

    Use your solution to the previous exercise or the following classes as a starting point for this question:

    • Planet.java
    • SolarSystem.java

    You will not need to change the Planet class but will be extending the SolarSystem class to include additional functionality.

    Add a ListByDistance() method to the SolarSystem class to return a copy of the planet is sorted by distance from the sun so that the first entry is Mercury, and the last is Neptune. You will need to define a class that implements the Comparator<Planet> interface. Write a test case for your class.

    Add a second SolarSystem.listByDiameter() method and have this use an anonymous class (of type Comparator<Planet>) to return a copy of the planet list sorted by diameter. Write a test case for this method.

  2. Shape Refactoring

    In the previous exercises on inheritance you created a hierarchy of shapes sub classes a Shape super class. This question asks you to refactor your solution to use an interface to represent a shape.

    Use your solution to the previous exercise or the following classes as a starting point for this question:

    • Shape.java
    • Rectangle.java
    • TestShapes.java

    Refactor the shape class by extract the getColor() and getArea() methods into an interface and gave the Shape class implement this interface. If you rename the Shape class to ShapeBase and call the Interface Shape then you need only change the sub class name of the super class in sub class definitions (Rectangle et al) - all other code, such the test cases, should still work using the Shape type.

  3. Search Strategy

    The Strategy Design Pattern is a powerful technique that relies on interfaces. A strategy allows a developer to write a generic algorithm within a class whilst allowing consumers of that class to refine the algorithm to meet their specific requirements.

    A simple search strategy example can be applied to the Drawing class using in a previous exercise on inheritance by defining a suitable interface. If you have completed the previous question from this chapter you can use this as your starting point and include the following class:

    • Drawing.java

    If you haven't completed the previous question then use your classes from the drawing example from the inheritance chapter, or the following starter classes :

    • Shape.java
    • Triangle.java
    • Rectangle.java

    You are going to define an interface called Selector that will include a single method to decide is a shape should be selected. A suitable signature is: 

    boolean select(Shape shape);

    Extend the Drawing class to include a method that takes a Selector object and returns a list of those shapes that have been selected (ie. the select method returns true).

    Write a test case that adds sample shapes to the Drawing and then define a selector that will select some of these shapes (for example blue shapes with an area greater than 5.0) and verify that the correct number and total are of the shapes selected agrees wit your test data.
