REST Web Server - Questions

These exercises are designed to expand on the topics introduced in the REST Web Server chapter of the Java Development training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is to build a simple REST web server using Spring Boot.

These exercises build on the exercises in the previous Spring Boot chapter. If you have not yet configured a Spring boo application you should do so before starting these exercises.

  1. Spring Boot

    Using your Java Spring Boot project and support for the REST web server to your POM: 

    <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

    You will need a class with a main method to create the web server by calling the StringApplication.run() passing the current class as a parameter. This class should be annotated with @SpringBootApplicationFor example: 

    package solutions;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Main {

    public static void main(String[] args) throws Exception {
        SpringApplication.run(Main.class, args);
    }
}

    You can start the Spring Boot server by running this annotated main class. Ensure you can serve up static pages from the public folder by creating a new folder called src/web, in IntelliJ mark this as a Resources Root folder, and create the src/web/public folder for your static HTML files. A simple HTML5 index.html page would be:

    <!DOCTYPE html>
<html lang="en">
<head>
    <title>Exams Web Application</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" href="css/styles.css" type="text/css" >
</head>
<body>
<h1>Web Application</h1>
<p>Server started.</p>
</body>
</html>

  2. Spring Boot REST Server

    Using your Spring server from the previous question extend the service to have a single REST endpoint as follows:

    • http://localhost:8080/start - return the string Service running

    You will need to create a POJO class annotated with @RestController and provide a single method annotated with @RequestMapping("/start") to respond to the start URL.

  4. Number Fountain Web Service

    This exercise builds on your solution to the previous chapter with a number fountain generating odd numbers or the following start classes:

    • Fountain.java
    • OddFountain.java
    • application.properties

    Create a second REST controller class that has a URL request mapping of fountain and a single method that has an endpoint of /list/{n} where n is the count of the numbers to generate. You can test your web service manually using the URL:

    • http://localhost:8080/fountain/list/5

    which should display the value [1, 3, 5, 7, 9]. Call this web service gain with a count of 3:

    • http://localhost:8080/fountain/list/3

    And you should see the next three odd numbers: [11, 13, 15]
