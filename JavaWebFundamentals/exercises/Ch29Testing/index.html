Unit Testing - Questions

These exercises are designed to expand on the topics introduced in the Unit Testing chapter of the Java Development training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is to practice developing code using a Test Drievn Development approach where you develop the tests in parallel with the code, and start with each test case.

  1. Testing the Triangle class

    To start with you will need to add the JUnit and Hamcrest dependencies to your POM file: 

    <dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.12</version>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.hamcrest</groupId>
    <artifactId>hamcrest-all</artifactId>
    <version>1.3</version>
</dependency>

    Create your Java test files in a package in the src/test/java folder. Don't forget to include the following import in your test files: 

    import static org.junit.Assert.*;

    In a previous exercise you were asked to define a class to represent a triangle storing the lengths of the three sides of the triangle including methods to indicate if the triangle is:

    • isosceles (two sides of the same length)
    • equilateral (all sides the same length)
    • right angled (sum of the squares of the two shorter sides equals the square of the longest side)

    DO NOT use your existing class but start again with a new class called TriangleShape and develop this class in combination with JUnit test cases. Do this in the following stages:

    1. Create a TriangleShape class with a single constructor that takes a double[] parameter - do not create any attributes or add code to this constructor.
    2. Add a public List<Double> getSides() method to this class that returns an empty ArrayList<Double>.
    3. Create a test class to create a triangle with sides of [2,3,4] and test that the number of sides returned by the getSides() method is 3 and that the list contains values [2,3,4]. This test should fail. You will find the Hamcrest containsInAnyOrder() method useful for this test. For example: 
      List<String> collection = Lists.newArrayList("ab", "cd", "ef");
assertThat(collection, containsInAnyOrder("cd", "ab", "ef"));
    4. Update your class to make this test pass.
    5. Add a (failing) test to create a triangle with only two sides [2,3] and verify that an expected IllegalArgumentException is thrown.
    6. Update your class to make this test pass.
    7. Add a test to create a triangle with four sides [2,3] and verify that an expected IllegalArgumentException is thrown. If this test doesn't pass update your class to make it pass.
    8. Add an outline boolean isIsosceles() method that always returns false.
    9. Create a test for an isosceles triangle [3,3,2] and update your class to make this pass.
    10. Test non isosceles triangles [2,3,4] and [3,3,3] (equilateral is not isosceles).
    11. Create a test for an equilateral triangle [3,3,3] and update your class to make this pass.
    12. Test non equilateral triangles .
    13. Add a test for right angled triangles ([3,4,5]) and update your class to make this pass, test an isosceles triangle that is also right angled, and a non right angled triangle.
